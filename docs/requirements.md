# Requirements Specification

## 1. Project Goal

Develop a professional full-stack scientific calculator that demonstrates modern software engineering principles, clean architecture, object-oriented programming, and algorithmic problem solving.

The application features a React frontend, a Python FastAPI backend, and a custom mathematical expression parser implemented from scratch.

---

## 2. Minimum Viable Product (MVP)

### Functional Requirements

The calculator will:

- evaluate arithmetic expressions
- support addition, subtraction, multiplication, and division
- support parentheses
- support decimal numbers
- respect operator precedence
- support keyboard input
- display the current expression
- display the calculated result
- provide meaningful error messages for invalid expressions

### Non-Functional Requirements

The application must:

- provide a responsive and intuitive user interface
- separate frontend and backend responsibilities
- follow object-oriented design principles where appropriate
- be modular and maintainable
- include automated tests for critical functionality
- handle invalid input gracefully
- be deployable as a web application
- be designed to support future scalability

---

## 3. Future Features

These features are intentionally postponed and will be implemented in later versions:

- scientific functions (sin, cos, tan, log, sqrt, ...)
- calculation history
- memory operations
- themes
- graph plotting
- export functionality
- keyboard shortcuts
